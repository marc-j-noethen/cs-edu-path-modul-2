# Advanced Phishing and MFA

## Summary Based on the 80/20 Principle

### 1. What this topic is about
This topic explains how multi-factor authentication helps against account takeover, where weaker MFA methods can fail, and why phishing-resistant MFA matters.

### 2. Key ideas
- MFA improves security by requiring more than one authentication factor.
- SMS and email codes are better than passwords alone, but they are relatively weak because they can be intercepted, forwarded, or socially engineered.
- Push-based MFA can be abused through approval fatigue if users accept prompts they did not initiate.
- Phishing-resistant methods such as FIDO2/WebAuthn security keys provide stronger protection against credential phishing and adversary-in-the-middle attacks.

### 3. Tools and practical relevance
- Authenticator apps are commonly used to generate time-based one-time passwords.
- Hardware security keys are a practical example of phishing-resistant MFA.

### 4. Key takeaway
Not all MFA is equally strong. The best MFA methods are designed to resist phishing, replay, and user fatigue.

## Table 2: Technical Terms

| Term | Meaning |
|---|---|
| MFA | Authentication using two or more independent factors |
| TOTP | Time-based one-time password generated by an app |
| Push fatigue | Repeated approval prompts used to pressure a user into accepting one |
| SIM swapping | Fraudulent transfer of a phone number to another SIM card |
| Phishing-resistant MFA | MFA designed to prevent credential replay and fake-login interception |

| Term | Meaning |
|---|---|

| deny | reject a login request |
| intercept | capture data in transit |
