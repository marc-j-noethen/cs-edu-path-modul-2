# CS Edu Path Module 2

Module 2 moves from basic security concepts into practical cybersecurity work. The focus shifts toward reconnaissance, social engineering, web security, cryptography, exploitation, malware, monitoring, DFIR, and GRC.

## Overview

| Area | Content |
|---|---|
| Module goal | Build a practical understanding of offensive and defensive security concepts |
| Focus | Reconnaissance, web security, crypto, exploitation, malware, SIEM, DFIR, GRC |
| Structure | Weekly folders with topic summaries and exercise placeholders |
| Scope | 10 weekly folders and 40+ topic folders |

## Topic Map

### Offensive foundations
- Week 1: Ethical hacking, passive and active reconnaissance, service enumeration
- Week 2: Social engineering, passwords, sessions, tokens, phishing, MFA
- Week 3: Web hacking, SQL injection, XSS, broken authentication, access control

### Crypto and exploitation
- Week 4: Symmetric encryption, hashing, asymmetric crypto, Diffie-Hellman, TLS
- Week 5: Vulnerability discovery, Metasploit, misconfigurations, common software flaws
- Week 6: Post-exploitation, Active Directory, credential attacks, lateral movement

### Detection, forensics, and governance
- Week 7: Malware basics, static analysis, dynamic analysis, malware defense
- Week 8: Security monitoring, SIEM investigation, network monitoring, endpoint monitoring
- Week 9: DFIR, disk forensics, memory forensics, network forensics
- Week 10: GRC, risk management, compliance, governance, policy

## Typical Unit Structure

- `1-Concepts-Tools-Terms/`
  Concise summaries of core ideas, tools, technical terms, and key vocabulary.
- `2-Excersises/`
  Exercise folders or placeholders related to the topic. In this repository, many of these files are currently empty placeholders.

The folder name `2-Excersises` is preserved as-is even though the spelling is inconsistent.

## What You Should Learn

By the end of Module 2, you should be able to:

- explain the purpose and limits of reconnaissance, enumeration, and web testing
- understand common web, Windows, and software vulnerability classes at a high level
- distinguish core cryptographic building blocks and their security goals
- place malware, monitoring, and forensics concepts into a practical incident context
- understand why governance, risk, and policy matter alongside technical controls

## How To Use This Repository

- Work through the weeks in order when possible, because later topics build on earlier ones.
- Read the summary in `1-Concepts-Tools-Terms` before starting any lab or exercise.
- Use the exercise folders for your own notes, walkthroughs, or screenshots if needed.
- Treat all offensive-security topics as authorized lab material only.

## Position in the Overall Path

Module 2 is the bridge between security fundamentals and applied security practice. It prepares the ground for Module 3, where the same thinking is applied to cloud, enterprise, and DevSecOps environments.
