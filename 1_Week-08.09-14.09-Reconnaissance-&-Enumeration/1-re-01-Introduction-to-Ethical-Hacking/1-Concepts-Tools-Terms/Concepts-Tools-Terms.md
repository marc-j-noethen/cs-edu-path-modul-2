# Introduction to Ethical Hacking

## Summary Based on the 80/20 Principle

### 1. What this topic is about
This topic introduces ethical hacking, responsible security testing, and the CIA triad as a foundation for security thinking.

### 2. Key ideas
- Ethical hacking is only legitimate with explicit authorization, defined scope, and legal approval.
- The CIA triad describes three core security goals: confidentiality, integrity, and availability.
- Attackers, defenders, vulnerabilities, and controls are easier to understand when you think in systems rather than isolated tools.
- A strong ethical and legal foundation matters more than any single hacking technique.

### 3. Tools and practical relevance
- TryHackMe provides safe training environments for learning hands-on security concepts.
- Kali Linux is a common lab platform because it includes many security tools in one place.
- OpenVPN is often used to connect securely to isolated training labs.

### 4. Key takeaway
Ethical hacking starts with permission, scope, and purpose. The goal is to improve security, not to abuse access.

## Table 2: Technical Terms

|---|---|
